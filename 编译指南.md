## 环境准备

### 选项一：云端编译（无需安装任何软件）

#### 所需工具：
- GitHub 账号（免费）：https://github.com/
- 浏览器（用于访问GitHub）

#### 操作步骤：

1. **创建GitHub仓库**
   - 登录 GitHub：https://github.com/
   - 点击右上角 **"+"** → **"New repository"**
   - 填写信息：
     - Repository name: `voice-to-text`（或你喜欢的名字）
     - Description: `语音转文字 Android APP`
     - 选择 **Private**（私有）或 **Public**（公开）
     - **不要**勾选 "Add a README file"
     - 点击 **"Create repository"**

2. **上传代码到GitHub**

   **方法A：使用网页上传（最简单）**
   - 在新建的仓库页面
   - 点击 "uploading an existing file"
   - 把 `d:\Desktop\voice` 文件夹的所有内容拖拽到网页上
   - 点击 "Commit changes"

   **方法B：使用Git命令**
   ```powershell
   # 打开PowerShell，导航到项目目录
   cd d:\Desktop\voice

   # 初始化Git仓库
   git init

   # 添加所有文件
   git add .

   # 创建提交
   git commit -m "Initial commit: VoiceToText Android APP"

   # 添加远程仓库（替换 YOUR_USERNAME 为你的GitHub用户名）
   git remote add origin https://github.com/YOUR_USERNAME/voice-to-text.git

   # 推送代码
   git branch -M main
   git push -u origin main
   ```

3. **触发云端编译**

   - 在GitHub仓库页面，点击 **"Actions"** 标签
   - 你会看到 "Build Android APK" workflow
   - 点击 **"Run workflow"** 按钮
   - 在弹出框中：
     - 选择 "debug" 作为 APK 类型（推荐）
     - 点击绿色 **"Run workflow"** 按钮
   - 等待编译完成（通常5-10分钟）

4. **下载APK**

   - 编译完成后，点击该workflow运行记录
   - 在页面底部找到 **"Artifacts"** 部分
   - 点击 `debug-apk` 下载Debug版本
   - 或者点击 `release-apk` 下载Release版本
   - 下载的是一个ZIP文件，解压后得到 `.apk` 文件

#### 云端编译优势：

✅ **零配置** - 无需安装JDK、Android Studio等  
✅ **免费额度** - GitHub每月提供2000分钟免费编译时间  
✅ **自动环境** - 不需要担心JDK、SDK版本问题  
✅ **多平台支持** - Windows、Mac、Linux都可以使用  
✅ **版本管理** - 自动保存所有编译历史

## 本地编译

### 环境要求

#### 1. JDK 17+

**下载地址**：
- Adoptium (推荐): https://adoptium.net/temurin/releases/?version=17
- Oracle JDK: https://www.oracle.com/java/technologies/downloads/#java17

**验证安装**：
```powershell
java -version
# 应该显示：
# openjdk version "17.x.x"
```

#### 2. Android Studio

**下载地址**：https://developer.android.com/studio

**推荐版本**：Android Studio Hedgehog (2023.1.1) 或更高

**安装步骤**：
1. 下载安装包并运行
2. 按照向导完成安装
3. 首次启动会自动下载SDK组件
4. 等待下载完成

#### 3. Android SDK Platform 34

在Android Studio中配置：
1. 打开 File → Settings（Windows）或 Android Studio → Preferences（Mac）
2. 导航到 Appearance & Behavior → System Settings → Android SDK
3. 点击 "SDK Platforms" 标签
4. 勾选 "Android 14" (API 34)
5. 点击 "Apply" 下载

### 编译步骤

#### 使用Android Studio编译

1. **打开项目**
   - 启动 Android Studio
   - 选择 File → Open
   - 选择文件夹 `d:\Desktop\voice`
   - 点击 OK

2. **等待同步**
   - 首次打开会下载所有依赖
   - 底部状态栏显示进度
   - 耐心等待（可能需要10-20分钟）

3. **编译Debug APK**
   - 在右侧菜单点击 **Gradle** 图标（象鼻🐘图标）
   - 展开 app → Tasks → build
   - 双击 **assembleDebug**
   - 底部 "Build" 窗口显示编译进度

4. **找到APK**
   - File → Open In → Explorer（Windows）或 Finder（Mac）
   - 导航到 `app/build/outputs/apk/debug/`
   - 找到 `app-debug.apk` 文件

#### 使用命令行编译

1. **打开终端**
   - Windows: PowerShell 或 CMD
   - Mac/Linux: Terminal

2. **导航到项目目录**
   ```powershell
   cd d:\Desktop\voice
   ```

3. **编译Debug APK**
   ```powershell
   ./gradlew assembleDebug
   ```

4. **编译Release APK**
   ```powershell
   ./gradlew assembleRelease
   ```

5. **清理并重新编译**
   ```powershell
   ./gradlew clean assembleDebug
   ```

### APK位置

- **Debug版本**: `app/build/outputs/apk/debug/app-debug.apk`
- **Release版本**: `app/build/outputs/apk/release/app-release.apk`

## APK签名

### 什么是APK签名？

APK签名用于验证APK的开发者身份，确保：
- APK未被篡改
- 来自合法的开发者

### Debug签名（自动生成）

Android Studio会自动为Debug版本生成调试签名：
- 位置：`~/.android/debug.keystore`
- 密码：`android`
- 别名：`androiddebugkey`

### Release签名（需要手动配置）

#### 方法1：使用Android Studio生成

1. **生成签名密钥**
   - 打开 Android Studio
   - 点击 Build → Generate Signed Bundle / APK
   - 选择 APK → Next
   - 点击 "Create new"
   - 填写信息：
     - Key store path: 选择保存位置
     - Password: 设置密码（记住它！）
     - Confirm: 确认密码
     - First and Last Name: 你的名字
     - Organization: 公司/组织
     - City: 城市
     - Country: 国家代码（如 CN）
   - 点击 OK

2. **导出签名密钥**
   - 保存密钥库文件（`.jks` 或 `.keystore`）
   - **重要**：妥善保管这个文件，丢失将无法更新应用！

3. **配置签名**
   - 在 `app/build.gradle` 中添加：
   ```groovy
   android {
       signingConfigs {
           release {
               storeFile file('your-keystore.jks')
               storePassword 'your-password'
               keyAlias 'your-alias'
               keyPassword 'your-key-password'
           }
       }
       buildTypes {
           release {
               signingConfig signingConfigs.release
           }
       }
   }
   ```

#### 方法2：使用命令行生成

```powershell
# 生成签名密钥
keytool -genkey -v -keystore my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key-alias

# 参数说明：
# -keystore: 密钥库文件名
# -keyalg: 密钥算法（RSA）
# -keysize: 密钥长度（2048位）
# -validity: 有效期（天数）
# -alias: 密钥别名
```

#### 方法3：使用GitHub Secrets（推荐用于云编译）

1. **生成签名密钥**
   ```powershell
   keytool -genkey -v -keystore signing.jks -keyalg RSA -keysize 2048 -validity 10000 -alias signing-key -storepass your-storepass -keypass your-keypass -dname "CN=Your Name, O=Your Organization, C=CN"
   ```

2. **转换为Base64**
   ```powershell
   # Windows
   [Convert]::ToBase64String([System.IO.File]::ReadAllBytes("d:\Desktop\voice\signing.jks")) > base64.txt

   # Mac/Linux
   base64 signing.jks > base64.txt
   ```

3. **添加GitHub Secrets**
   - 进入 GitHub 仓库
   - 点击 Settings → Secrets → Actions
   - 添加以下 Secrets：
     - `KEYSTORE_FILE`: base64.txt 文件内容
     - `KEYSTORE_PASSWORD`: 密钥库密码
     - `KEY_ALIAS`: 密钥别名
     - `KEY_PASSWORD`: 密钥密码

4. **配置Workflow**
   - GitHub Actions 会自动使用这些密钥签名

## 常见问题

### Q1: 编译失败，提示 "Could not find gradle"

**原因**：gradle-wrapper.jar 文件未下载或损坏

**解决方案**：
```powershell
# 删除旧的wrapper
Remove-Item -Recurse -Force gradle\wrapper

# 重新创建wrapper
gradle wrapper --gradle-version=8.2
```

或者手动下载 gradle-wrapper.jar：
```powershell
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/gradle/gradle/v8.2.0/gradle/wrapper/gradle-wrapper.jar" -OutFile "gradle\wrapper\gradle-wrapper.jar"
```

### Q2: 编译失败，提示 "SDK location not found"

**原因**：ANDROID_HOME 环境变量未设置或SDK路径错误

**解决方案**：

**Windows：**
```powershell
# 设置环境变量（临时）
$env:ANDROID_HOME = "C:\Users\YourUserName\AppData\Local\Android\Sdk"

# 或永久设置
[Environment]::SetEnvironmentVariable("ANDROID_HOME", "C:\Users\YourUserName\AppData\Local\Android\Sdk", "User")
```

**Mac/Linux：**
```bash
export ANDROID_HOME=~/Library/Android/sdk
```

**Android Studio中：**
1. File → Settings → Appearance & Behavior → System Settings → Android SDK
2. 记录 SDK Location 路径
3. 设置环境变量

### Q3: 编译失败，提示 "Failed to resolve: com.google.dagger:hilt-android"

**原因**：依赖下载失败或网络问题

**解决方案**：
1. 检查网络连接
2. 清理缓存并重新编译：
   ```powershell
   ./gradlew clean
   rm -rf ~/.gradle/caches
   ./gradlew assembleDebug
   ```
3. 使用国内镜像（修改 build.gradle）：
   ```groovy
   allprojects {
       repositories {
           maven { url 'https://maven.aliyun.com/repository/google' }
           maven { url 'https://maven.aliyun.com/repository/public' }
           maven { url 'https://maven.aliyun.com/repository/gradle-plugin' }
       }
   }
   ```

### Q4: APP闪退，提示 "App not installed"

**原因**：
- 签名冲突（已安装相同签名的旧版本）
- USB调试未启用

**解决方案**：
1. 卸载手机上的旧版本APP
2. 在手机上启用开发者选项：
   - 设置 → 关于手机 → 连续点击"版本号"7次
3. 启用USB调试：
   - 设置 → 开发者选项 → 启用USB调试

### Q5: APP崩溃，提示 "Java.lang.NoClassDefFoundError"

**原因**：ProGuard/R8混淆问题

**解决方案**：
1. 在 `app/build.gradle` 中禁用混淆：
   ```groovy
   android {
       buildTypes {
           debug {
               minifyEnabled false
           }
       }
   }
   ```

2. 或在 ProGuard 规则中添加：
   ```
   -keep class com.voicetotext.** { *; }
   -keepclassmembers class * {
       @com.google.gson.annotations.SerializedName <fields>;
   }
   ```

### Q6: GitHub Actions 编译失败

**查看日志**：
1. 点击失败的 workflow 运行
2. 查看每个步骤的日志输出
3. 找到错误信息

**常见问题**：

**问题**：编译超时
```
The workflow running time exceeded the limit
```
**解决**：检查代码是否有死循环或内存泄漏

**问题**：依赖下载失败
```
Could not resolve dependency: com.google.dagger:hilt-android
```
**解决**：在 workflow 中添加镜像源

**问题**：权限不足
```
Permission denied: 'gradlew'
```
**解决**：确保 gradlew 文件有执行权限
```powershell
git update-index --add --chmod=+x gradlew
```

### Q7: 如何更新APP？

1. **本地编译更新**：
   - 修改代码
   - 重新编译 `./gradlew assembleDebug`
   - 通过USB安装到手机：
     ```powershell
     adb install -r app/build/outputs/apk/debug/app-debug.apk
     ```
   - `-r` 参数会替换已安装的APP

2. **云端编译更新**：
   - 修改代码
   - 推送 到GitHub
   - 重新触发 Actions
   - 下载新的APK文件

### Q8: 如何生成发布版本？

**使用Android Studio**：
1. Build → Generate Signed Bundle / APK
2. 选择 APK
3. 选择或创建签名密钥
4. 选择 release build variant
5. 点击 Create

**使用命令行**：
```powershell
./gradlew assembleRelease
```

### Q9: 编译后APK太大怎么办？

**优化APK大小**：
1. 启用ProGuard/R8：
   ```groovy
   android {
       buildTypes {
           release {
               minifyEnabled true
               shrinkResources true
           }
       }
   }
   ```

2. 只保留需要的ABI架构：
   ```groovy
   android {
       splits {
           abi {
               enable true
               reset()
               include 'armeabi-v7a', 'arm64-v8a'
               universalApk true
           }
       }
   }
   ```

3. 使用App Bundle（AAB）发布：
   - Google Play会自动生成适合设备的APK
   - 显著减少APK大小

### Q10: 如何调试APK？

**使用Android Studio**：
1. 连接手机（启用USB调试）
2. 点击 Run → Debug
3. 设置断点
4. 调试运行

**使用命令行**：
```powershell
# 安装并调试
adb install -t app/build/outputs/apk/debug/app-debug.apk

# 查看日志
adb logcat

# 过滤APP日志
adb logcat -s com.voicetotext
```

## 高级配置

### 配置多个签名环境

```groovy
android {
    signingConfigs {
        debug {
            storeFile file('debug.keystore')
            storePassword 'android'
            keyAlias 'androiddebugkey'
            keyPassword 'android'
        }
        release {
            storeFile file('release.keystore')
            storePassword System.getenv('KEYSTORE_PASSWORD')
            keyAlias System.getenv('KEY_ALIAS')
            keyPassword System.getenv('KEY_PASSWORD')
        }
    }
    buildTypes {
        debug {
            signingConfig signingConfigs.debug
        }
        release {
            signingConfig signingConfigs.release
        }
    }
}
```

### 配置编译变体

```groovy
android {
    flavorDimensions += "environment"
    productFlavors {
        dev {
            dimension "environment"
            applicationIdSuffix ".dev"
            versionNameSuffix "-dev"
        }
        prod {
            dimension "environment"
        }
    }
}
```

## 性能优化

### 加快编译速度

1. **启用Gradle缓存**：
   ```properties
   # gradle.properties
   org.gradle.caching=true
   org.gradle.parallel=true
   ```

2. **分配更多内存**：
   ```properties
   org.gradle.jvmargs=-Xmx4g -XX:+HeapDumpOnOutOfMemoryError
   ```

3. **使用增量编译**：
   ```properties
   android.enableJetifier=true
   android.useAndroidX=true
   ```

### 优化APK性能

1. **移除未使用的资源**：
   ```groovy
   android {
       buildTypes {
           release {
               shrinkResources true
               minifyEnabled true
           }
       }
   }
   ```

2. **使用ProGuard**：
   ```properties
   # 混淆规则
   -keep class com.voicetotext.data.remote.** { *; }
   -keep class com.voicetotext.data.local.** { *; }
   ```
